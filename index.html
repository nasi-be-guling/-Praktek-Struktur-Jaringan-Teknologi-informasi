Bagian 1

Logika "Di Balik Tombol" & Konsep CRUD

Mitos "Tombol Ajaib"

Yang Dilihat Perawat

Perawat di poli mengisi formulir pendaftaran di layar komputer (Front-End).

Setelah selesai, ia mengklik tombol biru besar bertuliskan "SIMPAN DATA". Tiba-tiba data tersimpan. Seolah-olah ada sihir (magic) di dalam tombol tersebut.

Yang Terjadi Sebenarnya

Tombol itu hanyalah pemicu (trigger). Saat diklik, aplikasi menerjemahkan isian formulir menjadi sebaris teks perintah rahasia.

Teks perintah inilah yang melesat melalui kabel LAN menuju Ruang Server Database untuk dieksekusi.

Apa itu SQL?

SQL (Structured Query Language) adalah bahasa standar internasional yang digunakan manusia dan aplikasi untuk "berbicara" dengan Database Relasional (seperti MySQL, Oracle, SQL Server).

Server Database tidak mengerti bahasa Indonesia atau klik mouse. Ia HANYA mengerti bahasa SQL.

Sebagai insinyur elektromedis/IT, Anda harus bisa menulis bahasa ini secara manual untuk mendiagnosis kerusakan data (Troubleshooting).

Jantung Aplikasi: C-R-U-D

Sebesar apapun aplikasi di dunia ini (SIMRS, Tokopedia, Instagram), 99% fungsi database-nya bermuara pada 4 operasi fundamental yang disebut CRUD.

C - Create

Membuat/Memasukkan baris data pasien baru.

R - Read

Membaca, memanggil, atau memfilter data.

U - Update

Memperbarui/mengedit data yang sudah ada.

D - Delete

Menghapus baris data dari database.

Pemetaan CRUD ke SQL

  • CREATE diterjemahkan menjadi perintah INSERT INTO
  • READ diterjemahkan menjadi perintah SELECT
  • UPDATE diterjemahkan menjadi perintah UPDATE ... SET
  • DELETE diterjemahkan menjadi perintah DELETE FROM

Info Penting

Bahasa SQL bersifat Case-Insensitive (Tidak peduli huruf besar/kecil). Namun, sebagai praktik terbaik (Best Practice), semua perintah utama SQL selalu ditulis dengan HURUF KAPITAL agar mudah dibedakan dengan nama tabel.

Bagian 2

Membedah Sintaks CREATE & READ

1. CREATE (Insert Data)

Digunakan saat perawat mendaftarkan pasien baru di Loket Pendaftaran.

Rumus Baku: Tentukan nama tabel, lalu sebutkan nama kolomnya, kemudian masukkan nilainya secara berurutan.

INSERT INTO pasien (no_rm, nama, gol_darah, umur) VALUES ('RM-001', 'Budi Santoso', 'O', 45);

Aturan Tipe Data

Nilai bertipe Teks (String) WAJIB diapit tanda kutip satu ('Budi'). Sedangkan nilai bertipe Angka/Numerik TIDAK PERLU tanda kutip (45).

Hasil Eksekusi INSERT

Jika query sebelumnya dieksekusi, database akan menambahkan satu baris (record) baru di bagian paling bawah tabel pasien.

no_rm nama gol_darah umur
RM-001 Budi Santoso O 45
RM-002 Siti Aminah A 32

2. READ (Select Data)

Ini adalah query yang paling sering digunakan di rumah sakit. Digunakan untuk menampilkan atau memanggil data dari gudang ke layar komputer dokter.

Menampilkan SEMUA Kolom

Gunakan simbol bintang (asterisk *)

SELECT * FROM pasien;

Menampilkan Kolom SPESIFIK

Sebutkan nama kolom yang diinginkan

SELECT nama, gol_darah
FROM pasien;

Klausa WHERE (Filter)

Bagaimana jika tabel Anda memiliki 500.000 data pasien, dan Dokter hanya ingin mencari berkas pasien bernama "Anton"?

Menarik setengah juta data (SELECT *) akan membuat komputer hang (crash). Kita harus menggunakan Saringan (Filter).

Dalam SQL, saringan tersebut bernama klausa WHERE (Di mana).

Contoh Penggunaan WHERE

Operator yang bisa digunakan: = (sama dengan), > (lebih dari), < (kurang dari), AND (dan), OR (atau).

-- Mencari pasien bergolongan darah O SELECT nama, umur FROM pasien WHERE gol_darah = 'O';
-- Mencari pasien gol. darah A yang umurnya di atas 50 tahun SELECT * FROM pasien WHERE gol_darah = 'A' AND umur > 50;

Bagian 3

Membedah Sintaks UPDATE & DELETE

3. UPDATE (Edit Data)

Digunakan ketika ada kesalahan ketik nama, atau pasien pindah alamat, sehingga data harus diperbarui.

Rumus Baku: UPDATE (tabelnya) SET (kolom yang mau diubah) WHERE (kondisi pasien mana yang mau diubah).

UPDATE pasien SET umur = 46, gol_darah = 'AB' WHERE no_rm = 'RM-001';

ANCAMAN BENCANA!

Apa yang terjadi jika Anda lupa mengetikkan klausa WHERE di akhir perintah UPDATE?
UPDATE pasien SET umur = 46;

HASILNYA: SEMUA (100.000+) PASIEN DI RUMAH SAKIT AKAN BERUBAH UMURNYA MENJADI 46 TAHUN SECARA BERSAMAAN! DATA HANCUR!

Penyelamat: Primary Key

Untuk menghindari bencana salah ubah data (salah alamat), klausa WHERE pada UPDATE HARUS menggunakan penunjuk yang unik mutlak.

Jangan pernah menggunakan WHERE nama = 'Budi' (Karena bisa jadi ada 50 orang bernama Budi di RS).

Gunakan selalu Primary Key (Kunci Utama) seperti Nomor Rekam Medis (No_RM) sebagai target sasaran tembak UPDATE Anda!

4. DELETE (Hapus Data)

Digunakan untuk membuang baris data secara permanen dari database. Sama berbahayanya dengan UPDATE jika lupa memakai WHERE!

DELETE FROM pasien WHERE no_rm = 'RM-002';

Best Practice Rumah Sakit

Di dunia kesehatan sesungguhnya, data rekam medis haram hukumnya untuk di-DELETE permanen (Hard Delete) karena alasan riwayat hukum. Developer SIMRS biasanya hanya melakukan "Soft Delete" (Mengubah status kolom 'is_active' menjadi 0) agar data hilang dari layar tapi tetap ada di database.

Bagian 4

Skenario Praktikum (Lab XAMPP)

Cara Mengeksekusi Query

Buka http://localhost/phpmyadmin, klik database db_rssehat yang sudah Anda buat minggu lalu, lalu klik Tab SQL di bagian atas layar.

Aturan Main Praktikum

  • DILARANG KLIK TOMBOL GUI (Insert/Edit/Delete)!
    Hari ini Anda adalah Administrator Database sejati. Semua manipulasi data wajib diketik manual menggunakan perintah SQL di tab "SQL".
  • Gunakan Titik Koma (;)
    Akhiri setiap perintah SQL Anda dengan tanda ; sebagai penutup baris.
  • Hati-Hati Typo!
    Satu huruf salah ketik pada nama tabel akan menghasilkan pesan Error berwarna merah (Syntax Error).

Misi 1: Registrasi 3 Pasien

Ketikkan kode berikut di tab SQL lalu klik tombol "Go" di pojok kanan bawah. Ini akan melakukan Multiple Insert (Memasukkan 3 data sekaligus).

INSERT INTO pasien (no_rm, nama, umur, gol_darah) VALUES ('RM-111', 'Ahmad Dani', 35, 'O'), ('RM-222', 'Susi Susanti', 28, 'AB'), ('RM-333', 'Joko Widodo', 60, 'B');

Misi 2: Pemanggilan Data

Tugas A: Tampilkan seluruh pasien yang ada di gudang data.

SELECT * FROM pasien;

Tugas B: Direktur hanya ingin melihat nama dan umur pasien yang umurnya di atas 30 tahun.

SELECT nama, umur FROM pasien WHERE umur > 30;

Misi 3: Revisi (Update)

Terjadi kesalahan petugas pendaftaran. Ternyata umur Ibu Susi Susanti (RM-222) bukanlah 28 tahun, melainkan 29 tahun. Perbaiki datanya!

UPDATE pasien SET umur = 29 WHERE no_rm = 'RM-222';

Setelah di-run, lakukan SELECT * FROM pasien; lagi untuk memastikan datanya berubah!

Misi 4: Penghapusan (Delete)

Data Bapak Joko Widodo (RM-333) ternyata adalah data tes (dummy) dan harus dihapus dari sistem rumah sakit. Eksekusi pemusnahan datanya!

DELETE FROM pasien WHERE no_rm = 'RM-333';

Cek kembali tabel Anda, baris data RM-333 seharusnya sudah musnah.
